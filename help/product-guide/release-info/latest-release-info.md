# [!DNL AEM Guides] Versions

[!DNL Adobe Experience Manager Guides] est une application déployée sur AEM. Il s’agit d’une puissante solution de gestion de contenu de composants (CCMS) de niveau entreprise qui permet la prise en charge native de DITA en Adobe Experience Manager, permettant aux AEM de gérer la création et la diffusion de contenu basé sur DITA.

Les packages AEM Guides sont disponibles en deux variantes : les versions UUID et Non-UUID.

## Versions UUID et non-UUID

Les principales différences entre les versions UUID et non-UUID sont les suivantes :

|  | Génération non-UUID | Création d’UUID |
|---|---|---|
| **Identification des ressources** | Toutes les ressources sont identifiées à l’aide du chemin d’accès de la ressource dans le référentiel. | Toutes les ressources sont identifiées à l’aide de leur UUID (ID unique généré par le système lors du premier téléchargement de la ressource). |
| **Création de référence** | Toutes les références de contenu sont créées en fonction de leurs chemins. | Toutes les références de contenu sont créées en fonction de leur UUID. |

### Avantages de la création d’UUID

* L’installation d’UUID est plus performante :
   * Les références sont indépendantes du chemin : le système de gestion des références connaît les liens car les références sont créées en fonction des UUID et non des chemins.
   * Les opérations de déplacement/mise à jour sont efficaces : les UUID restent identiques même si les ressources se déplacent vers un autre chemin d’accès dans le référentiel. Par conséquent, aucun traitement n’est nécessaire pour corriger les références entre les ressources lors des opérations de déplacement/mise à jour.
* La génération UUID est tournée vers l’avenir, car nous utilisons également ce cadre pour la configuration cloud de AEM Guides.


### Choisissez entre les deux versions

* Si vous êtes un nouveau client, nous vous recommandons d’utiliser UUID build.
* Si vous êtes un client existant, vous pouvez choisir de passer à la génération UUID, car la migration de la génération Non-UUID vers la génération UUID est désormais possible. Pour plus d’informations, consultez la *section Migration* de contenu non-UUID vers UUID dans les guides Installation **et configuration Adobe Experience Manager.**

>[!NOTE]
>
>* Les clients devront choisir entre le mode UUID et le mode non-UUID au moment de la première configuration (si vous avez besoin d’aide, veuillez contacter Customer Success Manager pour vous aider à prendre la décision en fonction de votre cas d’utilisation).
>* Lors de la mise à niveau d’une version de AEM Guides vers une version plus récente, les clients doivent s’assurer qu’ils choisissent le même mode (UUID / non-UUID) pour correspondre à leur mode existant. Une génération non-UUID ne doit pas être directement mise à niveau vers une version UUID. Le passage d’une création non-UUID à une génération UUID nécessite une migration de contenu.

**Mise à niveau des builds**

Lorsque vous effectuez une mise à niveau d’une ancienne version vers une version plus récente de , vous devrez peut-être exécuter des scripts de [!DNL AEM Guides]migration. Reportez-vous aux notes de mise à jour et à la documentation spécifique à la version pour obtenir des instructions de mise à niveau.

Tous les chemins de mise à niveau ne sont pas directement pris en charge. Par exemple, la mise à niveau directe vers la version 4.0 n’est possible qu’à partir de la version 3.8.
Si vous utilisez une version antérieure à 3.8, reportez-vous à la documentation spécifique à votre version pour obtenir des instructions [de mise à niveau, Archive](https://helpx.adobe.com/fr/xml-documentation-for-experience-manager/archive.html) d’aide.
Contactez votre responsable de succès client pour valider le chemin de mise à niveau.
